# Silver.Vy — интернет-магазин одежды

Веб-приложение интернет-магазина с каталогом товаров, корзиной, регистрацией/авторизацией пользователей и оформлением заказов с отправкой email-уведомлений (функционал заказа в рамках учебной практики подразумевает только отправку письма на электронную почту, указанную при регистрации). На данный момент сайт загружен на хостинг https://render.com/ по ссылке https://silvervy-backend.onrender.com/index.html и обладает там полным функционалом, в том числе отправкой писем через собсвенный домен silvervy.ru. При локальном запуске понадобится получение API-ключа https://resend.com/, что подразумевает наличие собственного почтового домена, или же исключение функционала отправки писем, что будет подробно разобрано ниже.

## Стек технологий

| Слой | Технология |
|------|-----------|
| Backend | Node.js, Express.js |
| БД | PostgreSQL + Sequelize ORM |
| Аутентификация | JWT (jsonwebtoken) + bcryptjs |
| Email | Resend API |
| Frontend | HTML, CSS, JavaScript (статические файлы) |

## Установка и локальный запуск

### 1. Клонировать репозиторий

```bash
git clone https://github.com/<ваш-аккаунт>/<репозиторий>.git
cd <репозиторий>
```

### 2. Установить зависимости

```bash
npm install
```

### 3. Создать базу данных PostgreSQL

Запустите `psql` (или pgAdmin) и выполните:

```sql
CREATE DATABASE silvervy_db;
```

> Если хотите использовать другое имя — запомните его для следующего шага.

### 4. Настроить переменные окружения

Скопируйте файл-шаблон и заполните своими данными:

```bash
cp .env.example .env
```

Откройте `.env` и заполните:

```env
DB_HOST=localhost
DB_NAME=silvervy_db        # имя БД из шага 3
DB_USER=postgres            # ваш пользователь PostgreSQL
DB_PASSWORD=ваш_пароль

JWT_SECRET=любая_случайная_строка_32_символа

# Для отправки email при заказе (необязательно для проверки):
RESEND_API_KEY=re_xxxxxxxxxxxxxxxx
RESEND_FROM_EMAIL=info@silvervy.ru

NODE_ENV=development
PORT=5000
```

> **Без RESEND_API_KEY** сайт работает полностью, кроме отправки письма при оформлении заказа — в этом случае оформление заказа вернёт ошибку 500. Для полного тестирования функции заказов см. раздел «Email-уведомления» ниже.

### 5. Запустить приложение

```bash
node server/app.js
```

При успешном запуске вы увидите:

```
Подключение к PostgreSQL установлено
Database connection established
Server running on port 5000
```

> Sequelize автоматически создаст все таблицы (`Users`, `Carts`) при первом запуске.

### 6. Открыть в браузере

```
http://localhost:5000
```

## API-эндпоинты

### Аутентификация

| Метод | URL | Описание | Тело запроса |
|-------|-----|----------|-------------|
| POST | `/api/auth/register` | Регистрация | `{ name, email, password }` |
| POST | `/api/auth/login` | Вход | `{ email, password }` |

### Корзина *(требует JWT)*

| Метод | URL | Описание | Тело запроса |
|-------|-----|----------|-------------|
| GET | `/api/cart` | Получить содержимое корзины | — |
| PUT | `/api/cart` | Обновить корзину целиком | `{ items: [...] }` |

### Заказы *(требует JWT)*

| Метод | URL | Описание | Тело запроса |
|-------|-----|----------|-------------|
| POST | `/api/orders` | Оформить заказ | `{ city, address, notes? }` |

> JWT-токен передаётся в заголовке: `Authorization: Bearer <token>`

## Функциональность сайта

- **Главная страница** — видеофон, ключевые преимущества бренда
- **Каталог** — все товары с фильтрацией по категориям (верхняя одежда, низ, обувь)
- **Страницы товаров** — галерея фотографий, описание, выбор размера, кнопка «В корзину»
- **Корзина** — список выбранных товаров, итоговая сумма, форма оформления заказа
- **Регистрация / Вход** — JWT-аутентификация, данные хранятся в `localStorage`
- **Личный кабинет** — имя и email пользователя
- **О нас / Контакты** — информационные страницы

## Email-уведомления

При оформлении заказа на email покупателя отправляется письмо с деталями через сервис [Resend](https://resend.com). Для работы сервиса необходим свой домен, что уже реализованно на Render по ссылке из начала.

**Для тестирования без регистрации в Resend** — в `server/controllers/orderController.js` можно временно закомментировать блок `await resend.emails.send(...)` и раскомментировать строку ниже:

```js
// Временная заглушка (только для локального тестирования)
console.log('Order created:', orderNumber, 'for', user.email);
```

## Возможные ошибки при запуске

| Ошибка | Причина | Решение |
|--------|---------|---------|
| `password authentication failed` | Неверный пароль PostgreSQL | Проверьте `DB_PASSWORD` в `.env` |
| `database "silvervy_db" does not exist` | БД не создана | Выполните `CREATE DATABASE silvervy_db;` в psql |
| `connect ECONNREFUSED 127.0.0.1:5432` | PostgreSQL не запущен | Запустите службу PostgreSQL |
| `Error: RESEND_API_KEY is required` | Нет ключа Resend | Добавьте ключ в `.env` или см. раздел «Email-уведомления» |
